# AI部门月报共享版

这是一个适合两个人先试跑的共享网页版：

- 每个人以自己的身份单独记录 AI 工作
- 数据统一保存到一个本地 SQLite 数据库
- 页面里可以看到团队总信息表
- 可以按员工或按团队生成月报与绩效初评
- 支持把某个月的月报归档保存

## 不同网络访问

如果你们不在同一个局域网，就不能使用：

- `file:///...`
- `http://127.0.0.1:8000`
- `http://你的局域网IP:8000`

这种情况下，需要把当前项目部署到公网，拿到一个正式网页链接。

我已经把项目整理成可以直接部署的版本，并新增了：

- [render.yaml](E:\Work\Documents\GEO搜索\render.yaml)
- [requirements.txt](E:\Work\Documents\GEO搜索\requirements.txt)

最适合当前测试阶段的方案是部署到 `Render` 这类云平台，部署后会得到一个类似：

`https://你的项目名.onrender.com`

的公网链接，任何网络下都能打开。

## 启动方式

直接运行：

```powershell
.\start_shared_app.ps1
```

或者手动运行：

```powershell
& 'C:\Users\huawei\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe' .\server.py
```

启动后：

- 你自己打开 `http://127.0.0.1:8000`
- 同事在同一局域网下打开 `http://你的电脑IP:8000`

- 还没有账号密码和权限控制
- 只适合小范围测试
- 如果部署到云平台，当前 SQLite 数据库更适合测试，不适合正式长期存储
- 后续正式使用建议改成云数据库

## 临时公网访问

如果你们不在同一个局域网，又暂时不想走 GitHub / Render，可以直接运行：

```powershell
.\start_public_tunnel.ps1
```

它会做三件事：

1. 自动下载 `cloudflared`
2. 启动本地共享服务
3. 生成一个临时公网网址

生成后，把这个公网网址发给同事即可。

注意：

- 这是临时地址，重启后可能变化
- 你的电脑需要保持开机
- 你的共享版服务和公网隧道都要保持运行
